EasyRoads3D V3 Beta Manual
 
 

 

Release Notes v3 Beta 7.4

 

Important

Important regarding importing package updates

Do NOT remove the EasyRoads3D folder before importing a package update. The EasyRoads3D folder, includes prefabs with project info such as road types, side objects and also your newly created dynamic and custom connection prefabs. When you remove the EasyRoads3D folder you will lose these assets. Also close the Custom Prefab Editor window if currently open before importing a package update.

It is also recommended to switch road networks back to Edit Mode before importing a package update.

Please always backup your project before importing a package update.

 

Important regarding terrain objects

EasyRoads3D can make a backup of your terrain(s) and restore the terrain(s) in case something went wrong.

For additional safety measures It is recommended to duplicate the terrain object in the project panel so you always have a full back up at hand. Make sure to repeat this process after you changed the terrain!

This also applies to runtime scripts using the API to generate roads. Although this should work just fine, see the demo project runtime scene, it is always recommended to backup your terrains.

To back up your terrains simply duplicate the terrain object stored in the project panel. To find a reference to this terrain asset for the terrain in the scene, select the terrain in the scene and look in the Inspector: Terrain Collider Component > Terrain Data. Put focus on the terrain slot and the corresponding terrain will be highlighted in the project panel. Select it and do CTRL + D to duplicate it. Simply drag the duplicate terrain asset in the scene to replace the original if necessary.

 

Important regarding the Runtime API:

The following is only related to scripted road networks created with the runtime API. The below does not affect road networks manually created inside the editor.

In order to preserve the original terrain data, make sure you backup your terrains! In general restoring the terrains works well using RoadNetwork.RestoreRoadNetwork(), but it has occured that the terrain was not restored leaving the road shape in the terrain after exiting Play Mode. This particularly happened when you put focus on the scene view window while in Play Mode!

If you want to create a road network at runtime using the runtime API, it is recommended to check the above using a test terrain or make backups of your terrain(s).

 

Important regarding Side Objects and the river example in the demo project / video

In the video the terrain is manually smoothend out at the river edges. Note that in v3 manual terrain changes on areas not affected by roads or other deformation objects are preserved when switching back to Edit Mode. The outer part is not affected by the road, the manual changes will remain visiblein the terrain. So be careful with this, only smoothen out the terrain when you are happy with the river shape or when you do not mind to manually make corrections to the terrain after switching back to Edit Mode.

 

 

Feedback

This first beta builds are primarily intended for feedback on the following aspects:

1) The new workflow and the main functionality.

2) The new toolbar functionality.

3) Creating road types, creating roads in the scene, adding markers, inserting markers, deleting markers.

4) Creating dynamic prefabs, auto connecting prefas to roads, placing prefabs directly in the scene and attaching new roads to crossing connections. Changing crossing positions (select prefab and use the Q or W key to activate move and position gizmos / handles) and rotations (use the E key to activate the rotation gzimo). Changing the alignment priority connection (select the crossing prefab and click the grey handle associated with the connection you want to have alignment priority) and functionality.

6) handling prefabs in the scene such as changing prefab positions, rotations by using the road alignment option (is this useful or confusing?) .

To Do / Current limitations:

 

- Roundabouts: when adding connections, use the position slider to position it. When the current position overlaps another connection the roundabout will not be rebuild, a warning will appear in the console and the left and right side handles will turn red instead of green. When the current position overlaps the start or end index warning messages will appear as well! So stay within "save" ranges from the start (0 index) and end (depends on the resolution and radius).

- Roundabouts: changing the roundabout width or resolution after connections have already been added, may result in errors or the roundabout not being rebuild because connections are too close to eachother (see above). To fix this, select each roundabout connection from the dropdown in the Inspector and it should update automatically unless the connection overlaps another connection (see above).

- In a next build sidewalk and alignment handles will be visible without first having to select the prefab.

- In a next build the rotation alignment settings will be reset after manually changing the crossing rotation. Right now the manual changes will be lost after marker position changes to the road connected to the connection with alignment priority. To deal with this at the moment, you can turn off rotation alignment after manual changes to the prefab. You can do this by clicking the white active priority rotation handle so it turns grey.

- Circular road segments will be optimized further for smooth continuation of the road. For now you can achieve this by tweaking the position of the marker connected to the end of the circular segment.

- Tree /detail distance control per marker.

- "Preserve Terrain Shape" control per marker.
